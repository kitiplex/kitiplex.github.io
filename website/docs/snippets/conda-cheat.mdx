## Quickstart

:::tip[]

It is recommended to create a new environment for any new project or workflow.

:::


### Verify and check conda version

```sh
conda info
```
### Update conda in base environment

```sh
conda update --all -n base
```
```sh
conda update -n base conda
```
### Create a new environment

:::tip[]
Name environment descriptively. ie., `myenv`
:::



```sh
conda create --name myenv
```

```sh
conda activate myenv
```

## Packages

:::note[]

 Package dependencies and platform specifics are automatically resolved when using conda.

:::

### List installed packages.

```sh
conda list
```

### Update all packages

```sh
conda update --all
```
```sh
conda update --all -n base
```

```sh
conda update --all -n myenv
```
